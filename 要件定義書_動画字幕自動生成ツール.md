# 要件定義書：動画字幕自動生成ツール

**Moonshine ASR + 翻訳APIによるクラウド字幕生成パイプライン**

作成日: 2026年3月8日 | バージョン: 2.0

---

## 1. プロジェクト概要

- **プロジェクト名:** 動画字幕自動生成ツール (SubGen)
- **目的:** ダウンロードした動画から音声を抽出し、ASRで文字起こしを行い、スタイル付き字幕ファイルを自動生成する。クラウド上のWeb UIで操作可能
- **主なユースケース:** 日本語動画の日本語字幕作成 → 中国語翻訳字幕の生成。話者ごとの色分け対応のASS/SSA形式での出力を特徴とする

---

### 2.2 処理パイプライン

```
① 動画アップロード (MP4 → S3)
    ↓
② 音声抽出 (FFmpeg: MP4 → WAV 16kHz mono)
    ↓
③ 音声認識 (Moonshine Japanese → タイムスタンプ付きテキスト)
    ↓
④ 話者分離 (Speaker Diarization → 話者ID付与)
    ↓
⑤ 翻訳 (日本語 → 中国語 / DeepL or Google Translate API)
    ↓
⑥ 字幕ファイル生成 (SRT / ASS → S3)
    ↓
⑦ ダウンロード (S3 → ブラウザ)
```

## 3. 機能要件

### F-001: 動画アップロード・音声抽出

| 項目 | 内容 |
|------|------|
| 入力 | MP4ファイル（ブラウザからアップロード） |
| アップロード先 | Amazon S3 (`uploads/` プレフィックス) |
| 出力 | WAVファイル（16kHz, モノラル） |
| 使用ライブラリ | FFmpeg, boto3 |
| ファイルサイズ上限 | 2GB（Gradioデフォルト、設定で変更可能） |
| 備考 | アップロード時にプログレスバーを表示。処理完了後に一時ファイルを自動削除 |

### F-002: 音声認識（ASR）

| 項目 | 内容 |
|------|------|
| ASRエンジン | Moonshine 日本語モデル (moonshine-voice) |
| 入力 | 抽出済みWAV音声ファイル |
| 出力 | タイムスタンプ付きテキストセグメント (JSON) |
| 言語 | 日本語 (ja) |
| 処理方式 | バッチ処理（動画ファイル全体を一括処理） |
| 備考 | 長時間動画はチャンク分割して処理（30秒以下推奨）。処理進捗をWeb UIに表示 |

### F-003: 話者分離（Speaker Diarization）

| 項目 | 内容 |
|------|------|
| 機能 | 音声セグメントに話者IDを付与 |
| 候補ライブラリ | Moonshine内蔵のDiarization機能 / pyannote-audio |
| 出力 | 各セグメントに Speaker_1, Speaker_2 ... を付与 |
| 目的 | ASS字幕での話者ごとの色分けに使用 |

### F-004: 翻訳

| 項目 | 内容 |
|------|------|
| 翻訳API | DeepL API / Google Cloud Translation API（選択可能） |
| 対応方向 | **日本語 → 中国語**（メイン）、その他拡張可能 |
| 翻訳単位 | 字幕セグメント単位（タイムスタンプ保持） |
| 出力 | 原文（日本語）字幕 + 翻訳（中国語）字幕の両方を生成 |
| 備考 | APIキーは環境変数またはconfigファイルで管理 |

### F-005: 字幕ファイル生成

| 項目 | 内容 |
|------|------|
| 出力形式 | SRT（汎用） + ASS/SSA（スタイル付き） |
| SRT機能 | 標準的な番号・タイムスタンプ・テキスト形式 |
| ASS機能 | 話者ごとに異なる色を自動割り当て、フォント・サイズはデフォルト設定を提供 |
| エンコーディング | UTF-8 |
| ファイル命名 | `{入力ファイル名}_ja.srt` / `{入力ファイル名}_zh.ass` 等 |
| 保存先 | Amazon S3 (`outputs/` プレフィックス) |

### F-006: Web UI（Gradio）

| 項目 | 内容 |
|------|------|
| フレームワーク | Gradio |
| 画面構成 | 下記「Web UI画面仕様」参照 |
| アクセス | HTTPS（EC2のパブリックIP or ドメイン、ポート7860） |
| 認証 | Gradio内蔵のBasic認証（初期フェーズ） |
| レスポンシブ | Gradio標準のレスポンシブ対応 |

## 6. AWS構成詳細

### 6.1 EC2インスタンス

| 項目 | 推奨設定 |
|------|---------|
| インスタンスタイプ | t3.large（2vCPU, 8GB RAM）※Moonshine Tinyは軽量なので十分 |
| OS | Ubuntu 22.04 LTS |
| ストレージ | EBS 30GB（OS + モデル + 一時ファイル用） |
| セキュリティグループ | ポート7860 (Gradio), 22 (SSH) を開放 |
| Elastic IP | 固定IP付与（ドメイン設定用） |

### 6.2 S3バケット

| 項目 | 設定 |
|------|------|
| バケット名 | `subgen-{env}-files` |
| プレフィックス構成 | `uploads/` / `outputs/` / `temp/` |
| ライフサイクルルール | `temp/` は24時間で自動削除、`uploads/` は7日で自動削除 |
| アクセス | EC2からIAMロールでアクセス（アクセスキー不要） |

### 6.3 将来的な拡張構成

| 段階 | 構成 | 目的 |
|------|------|------|
| 初期 | EC2 + S3 | シンプル・低コスト |
| 中期 | + ALB + Route 53 | 独自ドメイン・HTTPS対応 |
| 将来 | + SQS + 複数EC2 | ジョブキューによる同時処理対応 |

## 8. ASS字幕スタイル仕様

話者ごとに自動で異なる色を割り当てる。デフォルトのカラーパレット:

| 話者 | 色（BGR） | 用途 |
|------|----------|------|
| Speaker 1 | `&H00FFFFFF` (白) | メインスピーカー用 |
| Speaker 2 | `&H0000FFFF` (黄) | サブスピーカー用 |
| Speaker 3 | `&H0000FF00` (緑) | 3人目以降 |
| Speaker 4 | `&H00FF8800` (水) | 4人目以降 |

デフォルト設定: フォント「Noto Sans JP」（中国語字幕は「Noto Sans SC」）、サイズ20、境界線付き、画面下部中央に配置。ユーザーがカスタマイズ可能な設定ファイルを提供予定。

## 10. 前提条件・依存関係

- AWSアカウントが利用可能であること
- EC2インスタンスの起動権限およびS3バケットの作成権限があること
- Python 3.10以上がEC2上にインストール可能であること
- FFmpegがEC2上にインストール可能であること
- moonshine-voiceパッケージがLinux環境で動作すること
- 翻訳機能使用時: DeepLまたはGoogle Cloud TranslationのAPIキー
- EC2のセキュリティグループでポート7860が開放されていること

## 11. リスク・課題

| リスク | 影響 | 対策 |
|--------|------|------|
| Moonshine日本語モデルの精度が不足 | 字幕の読みやすさが低下 | Whisperへのフォールバックを用意 |
| 話者分離の精度が不足 | 色分けが不正確になる | 話者数手動指定オプションを提供 |
| EC2のコスト | 常時起動でコスト発生 | 使わない時はインスタンス停止、将来的にスポットインスタンス検討 |
| 大容量動画のアップロード時間 | UXが低下 | プログレスバー表示、S3マルチパートアップロード対応 |
| 翻訳APIのコスト | 大量処理時にAPI費用が増加 | ローカルLLM翻訳も将来検討 |
| 日中翻訳の品質 | 専門用語や口語表現の翻訳精度 | 用語集（グロッサリー）機能を将来追加検討 |
| セキュリティ | 不正アクセスのリスク | Basic認証（初期）→ Cognito/IP制限（将来） |

## 12. デプロイ手順（概要）

```bash
# 1. EC2インスタンスにSSH接続
ssh -i key.pem ubuntu@<EC2_PUBLIC_IP>

# 2. システム依存パッケージのインストール
sudo apt update && sudo apt install -y ffmpeg python3.10 python3-pip

# 3. アプリケーションのセットアップ
git clone <REPO_URL> subgen && cd subgen
pip install -r requirements.txt

# 4. 環境変数の設定
cp .env.example .env
# .envに翻訳APIキー等を設定

# 5. Gradioサーバーの起動
python app.py --server-name 0.0.0.0 --server-port 7860

# 6. (オプション) systemdで常駐化
sudo cp subgen.service /etc/systemd/system/
sudo systemctl enable --now subgen
```
