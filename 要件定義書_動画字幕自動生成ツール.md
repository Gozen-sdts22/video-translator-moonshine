# 要件定義書：動画字幕自動生成ツール

**Moonshine ASR + 翻訳APIによるローカル字幕生成パイプライン**

作成日: 2026年3月8日 | バージョン: 1.1

---

## 1. プロジェクト概要

- **プロジェクト名:** 動画字幕自動生成ツール (SubGen)
- **目的:** ダウンロードした動画から音声を抽出し、ローカルASRで文字起こしを行い、スタイル付き字幕ファイルを自動生成する
- **主なユースケース:** 日本語動画の日本語字幕作成 → 中国語翻訳字幕の生成。話者ごとの色分け対応のASS/SSA形式での出力を特徴とする

---

## 2. システム構成（パイプライン）

```
① 動画入力 (MP4)
    ↓
② 音声抽出 (FFmpeg)
    ↓
③ 音声認識 (Moonshine Japanese)
    ↓
④ 話者分離 (Speaker Diarization)
    ↓
⑤ 翻訳 (日本語 → 中国語 / DeepL or Google Translate API)
    ↓
⑥ 字幕ファイル生成 (SRT / ASS)
```

---

## 3. 機能要件

### F-001: 動画からの音声抽出

| 項目 | 内容 |
|------|------|
| 入力 | MP4ファイル（ローカルパス指定） |
| 出力 | WAVファイル（16kHz, モノラル） |
| 使用ライブラリ | FFmpeg |
| 備考 | 動画の音声トラックを自動検出し、Moonshineが要求するサンプルレートに変換 |

### F-002: 音声認識（ASR）

| 項目 | 内容 |
|------|------|
| ASRエンジン | Moonshine 日本語モデル (moonshine-voice) |
| 入力 | 抽出済みWAV音声ファイル |
| 出力 | タイムスタンプ付きテキストセグメント (JSON) |
| 言語 | 日本語 (ja) |
| 処理方式 | バッチ処理（動画ファイル全体を一括処理） |
| 備考 | 長時間動画はチャンク分割して処理（30秒以下推奨） |

### F-003: 話者分離（Speaker Diarization）

| 項目 | 内容 |
|------|------|
| 機能 | 音声セグメントに話者IDを付与 |
| 候補ライブラリ | Moonshine内蔵のDiarization機能 / pyannote-audio |
| 出力 | 各セグメントに Speaker_1, Speaker_2 ... を付与 |
| 目的 | ASS字幕での話者ごとの色分けに使用 |

### F-004: 翻訳

| 項目 | 内容 |
|------|------|
| 翻訳API | DeepL API / Google Cloud Translation API（選択可能） |
| 対応方向 | **日本語 → 中国語**（メイン）、その他拡張可能 |
| 翻訳単位 | 字幕セグメント単位（タイムスタンプ保持） |
| 出力 | 原文（日本語）字幕 + 翻訳（中国語）字幕の両方を生成 |
| 備考 | APIキーは環境変数またはconfigファイルで管理 |

### F-005: 字幕ファイル生成

| 項目 | 内容 |
|------|------|
| 出力形式 | SRT（汎用） + ASS/SSA（スタイル付き） |
| SRT機能 | 標準的な番号・タイムスタンプ・テキスト形式 |
| ASS機能 | 話者ごとに異なる色を自動割り当て、フォント・サイズはデフォルト設定を提供 |
| エンコーディング | UTF-8 |
| ファイル命名 | `{入力ファイル名}_ja.srt` / `{入力ファイル名}_zh.ass` 等 |

| 項目 | 内容 |
|------|------|
| 実行環境 | Windows 10/11 ローカルPC |
| ランタイム | Python 3.10+ |
| UI形式 | CLI（コマンドライン）※将来Web UI拡張も視野に |

## 5. 技術スタック

| レイヤー | 技術 | 備考 |
|---------|------|------|
| 音声抽出 | FFmpeg | MP4からWAVへの変換。システムにインストールが必要 |
| 音声認識 | Moonshine (moonshine-voice) | pip install moonshine-voice。日本語モデルを自動DL |
| 話者分離 | Moonshine Diarization / pyannote-audio | Moonshineの内蔵機能を優先検証 |
| 翻訳 | DeepL API / Google Cloud Translation | APIキーが必要。日本語→中国語がメイン |
| 字幕生成 | Python (pysubs2 / 自作) | SRTとASS形式の両方を出力 |
| CLI | Python (argparse / click) | コマンドラインインターフェース |

## 6. CLI仕様（イメージ）

### 基本コマンド

```bash
# 基本使用（日本語動画 → 日本語字幕）
subgen input.mp4 --lang ja --format ass

# 翻訳付き（日本語動画 → 日本語字幕 + 中国語字幕）
subgen input.mp4 --lang ja --translate zh --api deepl

# 出力先指定 + SRTも同時出力
subgen input.mp4 -o ./subs/ --format both --translate zh
```

### CLIオプション一覧

| オプション | デフォルト | 説明 |
|-----------|-----------|------|
| `--lang, -l` | ja | 音声の言語（ja, en 等） |
| `--format, -f` | ass | 出力形式（srt / ass / both） |
| `--output, -o` | 入力と同じディレクトリ | 出力先ディレクトリ |
| `--translate, -t` | なし | 翻訳先言語（zh, en 等） |
| `--api` | deepl | 翻訳APIの選択（deepl / google） |
| `--speakers` | auto | 話者数（auto / 1 / 2 / 3 ...） |
| `--verbose, -v` | false | 詳細ログ出力 |

## 7. ASS字幕スタイル仕様

話者ごとに自動で異なる色を割り当てる。デフォルトのカラーパレット:

| 話者 | 色（BGR） | 用途 |
|------|----------|------|
| Speaker 1 | `&H00FFFFFF` (白) | メインスピーカー用 |
| Speaker 2 | `&H0000FFFF` (黄) | サブスピーカー用 |
| Speaker 3 | `&H0000FF00` (緑) | 3人目以降 |
| Speaker 4 | `&H00FF8800` (水) | 4人目以降 |

デフォルト設定: フォント「Noto Sans JP」（中国語字幕は「Noto Sans SC」）、サイズ20、境界線付き、画面下部中央に配置。ユーザーがカスタマイズ可能な設定ファイルを提供予定。

## 8. 開発フェーズ

| フェーズ | 内容 | ゴール |
|---------|------|--------|
| **Phase 1** | 基本パイプライン | MP4入力 → 音声抽出 → Moonshine ASR → SRT出力 |
| **Phase 2** | 話者分離 + ASS対応 | Diarization統合、話者色分けASS字幕出力 |
| **Phase 3** | 翻訳機能追加 | DeepL/Google API連携、日本語→中国語字幕生成 |
| **Phase 4** | UI拡張・品質改善 | Web UI化、字幕編集機能、精度チューニング |

---

## 9. 前提条件・依存関係

- Python 3.10以上がインストール済みであること
- FFmpegがシステムPATHに通っていること
- moonshine-voiceパッケージがインストール可能であること（Windows対応確認済み）
- 翻訳機能使用時: DeepLまたはGoogle Cloud TranslationのAPIキー
- 十分なディスク容量（モデルダウンロード用に数百MB）

## 10. リスク・課題

| リスク | 影響 | 対策 |
|--------|------|------|
| Moonshine日本語モデルの精度が不足 | 字幕の読みやすさが低下 | Whisperへのフォールバックを用意 |
| 話者分離の精度が不足 | 色分けが不正確になる | 話者数手動指定オプションを提供 |
| Windows環境での依存関係 | インストールが複雑化 | セットアップスクリプトを提供 |
| 翻訳APIのコスト | 大量処理時にAPI費用が増加 | ローカルLLM翻訳も将来検討 |
| 日中翻訳の品質 | 専門用語や口語表現の翻訳精度 | 用語集（グロッサリー）機能を将来追加検討 |
